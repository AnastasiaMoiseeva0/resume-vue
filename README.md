# Resume Generator — README

## Описание
Приложение "Генерация Резюме" — учебный проект на Vue.js. Позволяет редактировать блоки резюме (каждое поле — отдельный компонент), сохранять и отображать результат. Задание включает разбиение UI на компоненты, работу с emits, валидацию props/emits (дополнительно), динамические компоненты и интеграцию с Firebase.

## Требования (из задания)
1. Посмотреть видео "Что нужно делать".
2. Создать новое приложение с помощью Vue CLI.
3. Скачать исходники и заменить в своём проекте папку `src` на содержимое архива с исходниками.
4. Разбить приложение на компоненты и реализовать функционал как в видео.
5. Каждый отдельный элемент резюме (title, avatar, контакт, опыт и т.д.) — отдельный компонент.
6. Использовать emits для передачи событий от дочерних компонентов.
7. Разместить код на GitHub и прислать ссылку.
8. Задеплоить приложение на Firebase (посмотреть видео с инструкцией) и прислать ссылку.

## Функциональные требования
- Редактирование каждого поля резюме через UI (инлайн или в редакторе).
- Компоненты полей принимают данные через props и сообщают изменения через emits.
- Список блоков резюме может быть динамическим (использовать <component :is="componentName" v-bind="props" />).
- Сохранение/загрузка данных — локально и/или через Firebase (дополнительно).
- Валидация props и emits (дополнительная практика).

## Технические указания
- Используйте Vue CLI для создания проекта.
- Для динамических компонентов используйте component с передачей props через v-bind / this.$attrs.
- Для валидации props применяйте опцию props с типами и required; для emits — defineEmits (Composition API) или validate события вручную.
- Рекомендуется использовать Pinia или Vuex для хранения состояния резюме (необязательно).
- Для работы с Firebase:
  - Создайте проект в Firebase Console.
  - Настройте Firebase Hosting и/или Firestore/Realtime Database для хранения полей.
  - Добавьте конфигурацию Firebase в проект (не храните приватные ключи в публичных репозиториях — используйте переменные окружения).
  - Деплой на Firebase Hosting согласно официальной инструкции.

## Дополнительные задания (рекомендуется)
1. Полная валидация для всех props.
2. Полная валидация для emits.
3. Для списка блоков использовать <component :is="..."> и this.$attrs для приёма свойств.
4. Сохранять поля в Firebase и загружать их при старте.
